Origen del CSS

Programación16/10/2021

Voy a empezar explicando los orígenes tanto del CSS como del SCSS. Antes de la creación del CSS hubo dos creaciones antes las cuales fueron el CHSS creado por Håkon Wium Lie. La otra creación fue la de el SSP la cual la desarrolló Bert Bos.

Estos dos estilos fueron bastante famosos y bastante usados y sus dos creadores decidieron unirse para crear en 1994 el CSS. La unión Håkon Wium Lie y Bert Bos tenía como finalidad la creación de un nuevo lenguaje llamado CSS el cual se iba a basar en las creaciones de los dos cogiendo las mejores características de estas dos personas. En 1996 el W3C decidió apostar en el desarrollo y uso de este lenguaje y lo añadió a su grupo de trabajo.

A finales de 1996 el W3C publicó de forma oficial la primera recomendación de este lenguaje llamada CSS nivel 1. El 12 de Mayo de 1998 el grupo que se encargaba del CSS en la W3C publicó su segunda recomendación oficial de este lenguaje llamada CSS nivel 2.

Hoy en día,la versión de CSS que más se usa y más estandarizada es el CSS nivel 2.1. Este nivel de CSS es una continuación del nivel 2 cuya finalidad es la de ser una revisión del nivel 2. Aun así,este nivel 2.1 del CSS no se ha finalizado y sigue en desarrollo.Hasta nuestros días no se ha publicado una versión nueva de este lenguaje la cual sería llamada CSS nivel 3.

Origen del SCSS

Programación16/10/2021

En cuanto al SCSS es un pre-procesador el cual funciona mediante el uso del SASS usando la compatibilidad del CSS. Todo esto significa que el SCSS está basado en el SASS pero que muchas de las funcionalidades que se tiene implementadas tienen que ver con el uso del CSS.Debido a todo esto podemos decir que el SCSS no tiene un creador fijo ya que se basa entre el CSS y SASS.

El preprocesador de CSS es un lenguaje de hoja de estilos cuya función es la de permitirnos añadir nuevas funcionalidades relacionadas con el diseño de nuestra página.

Estos pre-procesadores no son muy necesarios ya que cualquier navegador está preparado para leer un CSS sin ningún pre-procesador pero en hojas de estilos que tengan cambios grandes van a necesitar un pre-procesador que consiga pasar toda esa información a CSS.
